# LINE メッセージ設計ルール

## Flex メッセージのレイアウト

### 重要情報は必ず先頭に置く

LINEのFlexメッセージはモバイル縦スクロールのため、末尾のコンテンツは読まれないまま閉じられる確率が高い。
「まとめ」「インサイト」「CTA」など読者に最も届けたいものはヘッダー直下に配置する。

```
❌ 誤（末尾に重要情報）:
  ニュース1, 2, 3... → まとめ（読まれない）

✅ 正（先頭に重要情報）:
  まとめ（ヘッダー内ボックスで強調）→ ニュース1, 2, 3... → CTA
```

### CTAボタンはフッターに固定する

予約・申し込みなどのコンバージョンボタンは `footer` コンポーネントに配置する。
`footer` は常に画面下部に固定表示されるため、スクロール位置に関わらず視認される。

## LINE からの離脱設計

### 外部リンクは離脱コストを生む

外部リンクタップ後にユーザーがLINEに戻り「まとめ」まで読む確率は低い。
WALOVERのターゲット（「信頼できる人に解釈を任せたい」層）にはリンク不要。

| 用途 | 推奨 |
|------|------|
| ニュースソース | リンクなし・ソース名のみ記載（信頼感を保ちつつ離脱なし） |
| 予約・申込 | LIFF URL（LINE内WebViewで完結） |
| 外部サービス | 原則使わない |

### テスト時は全体配信しない

broadcast() は本番配信のみ。開発・確認時はキーワードトリガーで
送信者本人のみに `replyMessage → pushMessage(userId)` で届ける。

## Claude API でメッセージを生成するときのトーン指定

### ブランド名を主語にしない

Claude への要約・生成プロンプトに「〇〇（社名）としては」という指示を入れると
宣伝口調の出力になる。LINEのフォロワーとの関係では「私たちとして」という
読者を含む一体感のある表現が適切。

```
❌ 誤（宣伝口調を招くプロンプト）:
「最後の1文はWALOVER（AI活用支援会社）の視点から見た示唆を書く」
→ 出力: 「WALOVERとしては、企業のAI導入支援にあたり〜」

✅ 正（共感口調を引き出すプロンプト）:
「私たちが引き続きAI活用を自分ごととしてキャッチアップしていくことが
大切という趣旨を自然な言葉で添える。ブランド名を主語にしない。」
→ 出力: 「引き続き自分ごととして動向をキャッチアップしていくことが重要です」
```

### Flex Message の color プロパティは必ず6桁HEXで指定する（2026-06-08 追記）

LINE API は3桁短縮カラーコードを不正値として拒否する。

```
❌ '#333'  '#555'  '#fff'  '#06c'
✅ '#333333'  '#555555'  '#ffffff'  '#0066cc'
```

エラー例: `{"message":"invalid property","property":"/body/contents/1/contents/0/color"}`

### ユーザーがコピーしたいテキストには clipboard アクションのボタンを付ける（2026-06-08 追記）

プロンプト・コード・URLなどコピペ前提のテキストは `clipboard` アクションで
ワンタップコピーを提供する（LINE App 9.19.0+）。

```typescript
{
  type: 'button',
  style: 'secondary',
  height: 'sm',
  action: {
    type: 'clipboard',
    label: 'コピー',
    clipboardText: textToСopy,  // コピーされるテキスト本文
  },
}
```

テキストボックスの直下に配置するのが最もわかりやすい。

### Claude 出力を構造化する場合はパーサー前提で設計する

複数のセクション（まとめ・ニュース一覧等）を1回の API 呼び出しで取得するとき、
出力フォーマットをパース可能な形式にして受け取り側で分解する。
プレーンテキストをそのまま Flex に流し込むと視覚的階層が作れない。

```typescript
// ❌ 誤: プレーンテキストをそのままテキストコンポーネントに入れる
{ type: 'text', text: summary, wrap: true }

// ✅ 正: セクション分割フォーマットでClaudeに出力させ、パーサーで分解する
// Claude 出力: [SUMMARY]\nまとめ\n\n[NEWS]\n絵文字|タイトル|要点
const { summary, items } = parseAiNewsSections(raw)
// → ヘッダー内ボックス・区切り線付きリストに当て込む
```
