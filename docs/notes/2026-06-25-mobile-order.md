# LINEで「言葉の壁」も「レジの行列」もなくす。沖縄の小さなお店のための注文システムを作りました

## レジに並ばせない、人を増やさない、言葉で困らせない

うるま市でAI活用支援をしているWALOVERです。今回は、飲食店向けの「モバイルオーダー」を自分たちのLINEシステムに組み込んだお話を、なるべく専門用語を使わずに書きます。

きっかけは単純で、「忙しい時間帯に注文を取りに行く人手が足りない」「レジ前で会計待ちの列ができる」「来てくれたお客さんと、その後つながれない」。小さなお店ほど抱えがちな、この悩みをまとめて解けないか、と考えたことでした。そして沖縄でお店をやるなら避けて通れない、もう一つの壁——**言葉**——にも答えを出したかった。

## できること

卓に貼ったQRをお客さんがスマホで読むと、**LINEの中で**メニューが開きます。アプリを入れる必要はありません。ここから、ひとつずつ機能を紹介します。

### 🍽 写真で選べる
メニューには**料理写真（イラスト）**が付いています。文字だけより断然わかりやすく、初めてのお客さんや、メニュー名にピンと来ない方でも「これ美味しそう」で選べます。見た目が華やぐと、注文のハードルもぐっと下がります。

### 🌏 英語対応＝「席で母国語で頼める」
画面のボタンひとつで、**メニューもボタンも英語に切り替わり**ます。LINEの言語設定から自動で判定もします。料理名や説明は、こちらが一品ずつ翻訳を打ち込むのではなく、**AIが自動で英訳**して裏側に貯めていく仕組みにしました。

地味ですが本質的なのが——**オーダーミスが消える**こと。お客さんが自分で英語のメニューを見て注文するので、「店員さんがうまく聞き取れず違う料理が出た」が起きません。沖縄の観光地のお店にとって、これは人手にも売上にも効きます。お客さんは英語で注文、**厨房の画面は日本語のまま**なので、スタッフはいつも通り作れます。

### 🧑‍🍳 厨房はテーブルごとに見える
注文は厨房の専用画面に、**テーブル単位でまとまって**届きます。提供したらワンタップで「提供済み」。ドリンクのように調理しないものまで「調理中」を挟まないよう、流れもシンプルにしました。

### 🧾 お会計は「お店が承認」してから
お客さんがLINEで「お会計をお願いする」を押すと、**勝手に会計済みにはならず**、お店の画面に「会計依頼」が届きます。スタッフが確認して承認して初めて確定。現金でもOKです。会計が終わった卓は画面からすっと消えて、見やすさを保ちます。

### 📊 「数字」がそのまま見える
ここがいちばん店長さんに喜ばれる部分かもしれません。1日の終わりに、お店の画面でこう見えます。

- 今日の売上と会計件数
- **平均滞在時間**（回転を考えるヒントに）
- **平均客単価**（卓あたり）

しかも滞在時間は、お客さんが**最初にメニューを開いた瞬間から**そっと計っています。「なんとなく忙しかった」を「今日は◯組・平均◯分・客単価◯円」に変える。勘と経験に、数字という味方を一枚足すイメージです。データは溜まっていくので、いずれ「週末は滞在が長い」「この時間は単価が高い」も見えてきます。

## いちばんの強みは「友だちが貯まる」こと

注文には**LINEの友だち追加が前提**です。来てくれた一見さんが、自然と「また案内を送れる相手」になる。クーポンや新メニューのお知らせが届く。**注文するたびに常連さんが貯まっていく**——これが、外部の大手アプリにはない、この仕組みのいちばんの芯です。

写真も英語対応も売上分析も、突き詰めれば「もう一度来てもらう」ための部品です。

## 作ってみて思うこと

派手な新機能を一度に積むより、現場の「これだと困る」を一つずつ形にする——その積み重ねでした。「会計はお店が承認したい」「写真が欲しい」「英語のお客さんが増えた」。聞いて、その日のうちに直して反映する。小さく速く回せるのが、自分たちで持っている強みだと改めて感じています。

沖縄には、ご家族や少人数で切り盛りしている素敵なお店がたくさんあります。人手を増やさずに「注文・会計・常連化」を一本につなぎ、言葉の壁まで越えられたら、もっと自分たちの時間を持てるはず。WALOVERは、そんな地元の現場に寄り添う道具を、これからも作っていきます。

気になった方は、お気軽に声をかけてください。
